MQTT Protocol - Complete IoT Reference Guide

=== WHAT IS MQTT? ===
MQTT (Message Queuing Telemetry Transport) is a lightweight, publish-subscribe network protocol designed for constrained devices and low-bandwidth, high-latency, or unreliable networks. It runs over TCP/IP and is the de facto standard for IoT messaging. MQTT was invented by Andy Stanford-Clark (IBM) and Arlen Nipper in 1999 and became an OASIS standard in 2014.

=== HOW MQTT WORKS ===
MQTT uses a broker-based pub/sub model:
1. Broker: Central server that receives all messages and routes them to subscribers (e.g., Mosquitto, HiveMQ, EMQX)
2. Publisher: Client that sends messages to a topic
3. Subscriber: Client that receives messages from subscribed topics
4. Topic: Hierarchical string used to filter messages (e.g., home/livingroom/temperature)

Message flow:
Device A publishes → Broker → All subscribers of that topic receive

=== TOPIC STRUCTURE ===
Topics are UTF-8 strings separated by forward slashes:
home/floor1/room1/temperature
factory/line3/machine7/status
sensors/outdoor/humidity

Wildcards:
+ (single level): home/+/temperature matches home/room1/temperature AND home/room2/temperature
# (multi level): home/# matches home/room1/temp AND home/room1/humidity AND home/room2/pressure

Reserved topics starting with $ are for broker use:
$SYS/broker/clients/connected - number of connected clients
$SYS/broker/messages/received - total messages received

=== QoS LEVELS ===
QoS 0 - At Most Once (Fire and Forget):
- Message sent once, no acknowledgment
- May be lost if broker is down or client disconnects
- Fastest, lowest overhead
- Use for: sensor telemetry where occasional loss is acceptable

QoS 1 - At Least Once:
- Message guaranteed to be delivered at least once
- Receiver sends PUBACK; sender retransmits if no ACK received
- May result in duplicate messages
- Use for: commands that must arrive but duplicates are safe

QoS 2 - Exactly Once:
- Four-part handshake: PUBLISH → PUBREC → PUBREL → PUBCOMP
- Guaranteed exactly once delivery, no duplicates
- Highest overhead
- Use for: billing data, financial transactions, critical commands

=== BROKER SETUP (MOSQUITTO) ===

Install Mosquitto on Ubuntu:
sudo apt-get install mosquitto mosquitto-clients
sudo systemctl enable mosquitto
sudo systemctl start mosquitto

Basic mosquitto.conf:
listener 1883
allow_anonymous true
log_type all
log_dest file /var/log/mosquitto/mosquitto.log

Secure configuration (with auth):
listener 8883
cafile /etc/mosquitto/certs/ca.crt
certfile /etc/mosquitto/certs/server.crt
keyfile /etc/mosquitto/certs/server.key
require_certificate false
allow_anonymous false
password_file /etc/mosquitto/passwd

Create password file:
sudo mosquitto_passwd -c /etc/mosquitto/passwd myuser

Test with CLI:
mosquitto_sub -h localhost -t home/# -v
mosquitto_pub -h localhost -t home/temp -m "23.5"

=== PYTHON PAHO-MQTT EXAMPLES ===

Install: pip install paho-mqtt

Basic Publisher:
import paho.mqtt.client as mqtt
import json
import time

BROKER = "broker.hivemq.com"
PORT = 1883
TOPIC = "iot/sensors/temperature"

client = mqtt.Client(client_id="esp32-sensor-01", protocol=mqtt.MQTTv5)

def on_connect(client, userdata, flags, rc, properties=None):
    if rc == 0:
        print("Connected to broker")
    else:
        print(f"Connection failed with code {rc}")

client.on_connect = on_connect
client.connect(BROKER, PORT, keepalive=60)
client.loop_start()

while True:
    payload = json.dumps({"temperature": 23.5, "humidity": 60, "device": "ESP32-01"})
    result = client.publish(TOPIC, payload, qos=1, retain=False)
    if result.rc == mqtt.MQTT_ERR_SUCCESS:
        print(f"Published: {payload}")
    time.sleep(5)

Basic Subscriber:
import paho.mqtt.client as mqtt

BROKER = "broker.hivemq.com"
PORT = 1883
TOPIC = "iot/sensors/#"

def on_connect(client, userdata, flags, rc, properties=None):
    print(f"Connected with result code {rc}")
    client.subscribe(TOPIC, qos=1)

def on_message(client, userdata, msg):
    print(f"Topic: {msg.topic}")
    print(f"Payload: {msg.payload.decode()}")
    print(f"QoS: {msg.qos}")
    print(f"Retain: {msg.retain}")

client = mqtt.Client(client_id="subscriber-01", protocol=mqtt.MQTTv5)
client.on_connect = on_connect
client.on_message = on_message

client.connect(BROKER, PORT)
client.loop_forever()

Publisher with TLS:
import paho.mqtt.client as mqtt
import ssl

client = mqtt.Client()
client.tls_set(
    ca_certs="/path/to/ca.crt",
    certfile="/path/to/client.crt",
    keyfile="/path/to/client.key",
    tls_version=ssl.PROTOCOL_TLSv1_2
)
client.username_pw_set("username", "password")
client.connect("secure-broker.example.com", 8883)

=== ARDUINO / ESP32 MQTT (PubSubClient) ===

#include <WiFi.h>
#include <PubSubClient.h>
#include <ArduinoJson.h>

const char* ssid = "YourSSID";
const char* password = "YourPassword";
const char* mqtt_server = "broker.hivemq.com";
const int mqtt_port = 1883;
const char* mqtt_topic_pub = "iot/esp32/sensors";
const char* mqtt_topic_sub = "iot/esp32/commands";

WiFiClient espClient;
PubSubClient mqttClient(espClient);

void callback(char* topic, byte* payload, unsigned int length) {
  String message = "";
  for (int i = 0; i < length; i++) {
    message += (char)payload[i];
  }
  Serial.println("Message on " + String(topic) + ": " + message);

  if (message == "restart") {
    ESP.restart();
  }
}

void reconnect() {
  while (!mqttClient.connected()) {
    String clientId = "ESP32-" + String(random(0xffff), HEX);
    if (mqttClient.connect(clientId.c_str())) {
      mqttClient.subscribe(mqtt_topic_sub);
      Serial.println("MQTT connected");
    } else {
      Serial.print("MQTT failed, rc=");
      Serial.println(mqttClient.state());
      delay(5000);
    }
  }
}

void publishSensorData(float temp, float hum) {
  StaticJsonDocument<128> doc;
  doc["device"] = "ESP32-01";
  doc["temperature"] = temp;
  doc["humidity"] = hum;
  doc["timestamp"] = millis();

  char buffer[128];
  serializeJson(doc, buffer);
  mqttClient.publish(mqtt_topic_pub, buffer, false);  // false = not retained
}

void setup() {
  Serial.begin(115200);
  WiFi.begin(ssid, password);
  while (WiFi.status() != WL_CONNECTED) delay(500);

  mqttClient.setServer(mqtt_server, mqtt_port);
  mqttClient.setCallback(callback);
  mqttClient.setKeepAlive(60);
  mqttClient.setBufferSize(512);
}

void loop() {
  if (!mqttClient.connected()) reconnect();
  mqttClient.loop();

  // Publish every 10 seconds
  static unsigned long lastMsg = 0;
  if (millis() - lastMsg > 10000) {
    lastMsg = millis();
    publishSensorData(23.5, 60.0);
  }
}

=== LAST WILL AND TESTAMENT (LWT) ===
LWT allows the broker to publish a message on behalf of a client if it disconnects unexpectedly:

// Python
client.will_set("devices/esp32-01/status", payload="offline", qos=1, retain=True)

// Arduino
mqttClient.connect(clientId, NULL, NULL, "devices/esp32-01/status", 1, true, "offline")

// On successful connection, publish "online" status
mqttClient.publish("devices/esp32-01/status", "online", true);

=== RETAINED MESSAGES ===
A retained message is stored by the broker and sent immediately to new subscribers:
mosquitto_pub -h localhost -t home/temp -m "23.5" -r   # -r = retain

Clear a retained message by publishing empty payload:
mosquitto_pub -h localhost -t home/temp -m "" -r

=== MQTT 5.0 FEATURES ===
MQTT 5.0 added:
- Session Expiry Interval: Auto-expire sessions
- Message Expiry Interval: TTL for messages
- Shared Subscriptions: Load balancing across multiple subscribers
  Subscribe to: $share/group_name/topic
- User Properties: Custom key-value headers
- Reason Codes: More detailed error codes
- Request/Response Pattern: Correlation data and response topic
- Topic Aliases: Reduce topic string overhead

=== MQTT RETURN/REASON CODES ===
0x00 - Connection accepted / Success
0x01 - Unacceptable protocol version
0x02 - Client identifier rejected
0x03 - Server unavailable
0x04 - Bad username or password
0x05 - Not authorized
0x81 - Malformed packet (MQTT 5.0)
0x87 - Not authorized (MQTT 5.0)
0x8D - Keep alive timeout (MQTT 5.0)
